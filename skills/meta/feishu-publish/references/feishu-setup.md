# 飞书自建应用创建指南

> 这一步只做一次。完成后凭证写到 `~/.feishu-credentials.json`，以后用脚本自动调。

## 为什么需要"自建应用"

飞书的 OpenAPI 不接受用户账号密码（出于安全），统一通过"应用"调用。
"自建应用"= 你为自己创建的一个机器人账号，专门给脚本用。

它的权限是你**主动授予**的，限定在你给它开的范围内（比如：只能改你授权的文档）。

---

## 一、创建应用

### 1. 打开开放平台

https://open.feishu.cn/app

用你的飞书账号登录。

### 2. 创建企业自建应用

主页 → **「创建应用」** → 选 **「企业自建应用」**。

> 如果选「商店应用」会进入提交审核流程，不要选那个。

### 3. 填基本信息

| 字段 | 填什么 |
| --- | --- |
| 应用名称 | `Markdown 文档推送器`（或者你想要的名字） |
| 应用描述 | `私人 markdown 推送工具` |
| 应用图标 | 上传一个或用默认 |

确认创建。

---

## 二、拿凭证

### 1. 进入应用详情

创建完会跳到应用详情页。如果没跳，从主页应用列表点进去。

### 2. 找凭证

左边菜单 → **「凭证与基础信息」** → 顶部 **「应用凭证」** 区域：

```
App ID:      cli_a1b2c3d4e5f6g7h8        [复制]
App Secret:  ··········· [显示] [复制]
```

把这两个值复制下来。

> ⚠️ App Secret 默认隐藏，点「显示」才能看到完整值。

---

## 三、申请权限

### 1. 进入权限管理

左边菜单 → **「权限管理」**。

### 2. 添加这些权限

> 点「**开通权限**」按钮，搜索权限名添加。

| 权限名 | 作用 |
| --- | --- |
| `docx:document` | 创建/读取/编辑云文档 |
| `docx:document:readonly` | （自动包含） |
| `drive:drive` | 文件操作（上传 markdown） |
| `drive:file` | 单文件操作（备用） |

确保以上四个权限都在「**已开通**」列表中。

### 3. 部分权限需要审核

`drive:drive` 这种"高敏感"权限可能需要管理员审批。如果你的飞书是**个人版**或**你自己的企业**，你就是管理员，会自动通过。

---

## 四、发布版本

> 这一步是关键。**不发布版本，应用调任何 API 都会报 99991672 错误**。

### 1. 进入版本管理

左边菜单 → **「版本管理与发布」**。

### 2. 创建版本

点 **「创建版本」**：

| 字段 | 填 |
| --- | --- |
| 应用版本号 | `1.0.0` |
| 版本说明 | `初始版本` |
| 更新说明 | `首次发布` |

提交。

### 3. 发布

提交后状态会变成 **「已通过」**（个人/自己企业）或 **「待审核」**（员工身份在公司租户）。

如果是「待审核」→ 联系管理员通过。
如果是个人版 → 应该立刻能用。

---

## 五、把应用加入文档协作者

> 这一步是告诉飞书："允许这个应用操作这篇文档。"
> 不做的话，应用调用会报 1061002 错误。

### 1. 打开目标文档

比如 `https://my.feishu.cn/docx/PBkudACpfomQhfxTLfGcyLylnwe`

### 2. 分享设置

右上角 **「分享」** → **「协作者」** 标签 → **「+ 添加协作者」**。

### 3. 搜应用名

在搜索框输入你创建的应用名（比如「Markdown 文档推送器」），下拉里会出现。

> 如果搜不到 → 检查应用版本是不是「已通过」。

### 4. 设置权限为「可编辑」

加进去，权限选 **「可编辑」**（默认是「可阅读」，要改）。

确认。

---

## 六、写凭证文件

### 1. 创建文件

在 Windows，打开记事本（或 VSCode），新建：

```json
{
  "app_id": "cli_a1b2c3d4e5f6g7h8",
  "app_secret": "XYZxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
  "default_doc_id": "PBkudACpfomQhfxTLfGcyLylnwe",
  "default_folder_token": ""
}
```

把 `app_id` 和 `app_secret` 替换成你刚才拿到的。

`default_doc_id` 是你想默认覆盖的文档 ID（飞书 URL `/docx/XXX` 那个 XXX）。

`default_folder_token` 是默认文件夹（留空 = 个人空间根目录）。

### 2. 保存为

路径：`%USERPROFILE%\.feishu-credentials.json`

完整路径示例：`C:\Users\你的用户名\.feishu-credentials.json`

> 文件名前面有个点 `.`，不要漏。Windows 资源管理器可能要勾选「显示隐藏文件」。

### 3. 验证

打开 PowerShell 跑：

```powershell
Test-Path "$env:USERPROFILE\.feishu-credentials.json"
# True 即可
```

---

## 七、测试连通

到 skill 目录下：

```bash
python scripts/publish.py --check
```

期望输出：

```
✓ 凭证有效
  app_id:  cli_a1b2c3d4e5f6g7h8
  token:   t-12345abc... (前 12 位)
  expires: 1700000000 (epoch)
```

如果失败：
- `99991663` → app_secret 错了，回 Step 二复制一遍
- `99991672` → 应用没发布，回 Step 四
- `网络错误` → 检查能不能访问 `open.feishu.cn`

---

## 错误码速查

| 错误码 | 含义 | 修复 |
| --- | --- | --- |
| 99991663 | app_secret 不对 | 重新复制凭证 |
| 99991668 | token 过期 | 脚本自动重试，无需操作 |
| 99991672 | 应用未发布 | 「版本管理与发布」点发布 |
| 1061002 | 无权操作文档 | 把应用加进文档协作者 |
| 1069902 | 文档不存在 | 检查 doc_id 是否正确 |

---

## 安全提示

- `~/.feishu-credentials.json` 文件**不要进 Git**
- `app_secret` 等同密码，**不要发到公共群、博客、GitHub**
- 怀疑泄露：飞书开放平台「凭证与基础信息」可以重置 secret
